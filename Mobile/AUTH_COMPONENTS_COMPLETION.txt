COMPONENT OVERVIEW

1. AuthLayout
   Purpose: Main wrapper for auth screens
   Features:
   - Centered content
   - Safe area aware
   - Keyboard aware
   - Hero image support
   - Form section support
   - Footer section support
   - Scrollable on small screens

2. LogoCard
   Purpose: Display logo inside white card with elevation
   Features:
   - Subtle shadow
   - Three size variants (small, medium, large)
   - Centered layout
   - Image resizing

3. PrimaryButton
   Purpose: Main CTA button with role-based color
   Features:
   - Loading spinner
   - Disabled state
   - Role-based colors
   - Three sizes (small, medium, large)
   - Icon support (future)

4. SecondaryButton
   Purpose: Outlined button variant
   Features:
   - Same as PrimaryButton
   - Outlined style (border only)
   - Uses role primary color for border

5. AppInput
   Purpose: Text input with comprehensive features
   Features:
   - Label above input
   - Error state with red border
   - Helper text below input
   - Icon prefix support
   - Password visibility toggle
   - PIN mode (numeric, spaced)
   - Focus color changes with role
   - Placeholder text
   - Disabled state

6. RememberMeCheckbox
   Purpose: Custom checkbox for remember me
   Features:
   - Checked/unchecked states
   - Custom label
   - Role-based accent color
   - Touch feedback

7. Typography
   Purpose: Consistent text styling
   Variants:
   - title (28px, bold, headings)
   - subtitle (20px, section heads)
   - body (16px, default text)
   - caption (14px, helper text)
   - button (16px, button text)
   - error (14px, red error text)
   Bold variants for body and caption

USAGE EXAMPLES

Basic Login:
```jsx
<AuthLayout
  hero={<LogoCard source={require('./logo.png')} />}
  form={
    <>
      <AppInput label="Email" value={email} onChangeText={setEmail} />
      <AppInput label="Password" password value={pass} />
      <PrimaryButton label="Sign In" onPress={handleLogin} />
    </>
  }
/>
```

Role Selection:
```jsx
{roles.map(role => (
  <SecondaryButton
    key={role}
    label={role}
    role={role}
    onPress={() => selectRole(role)}
  />
))}
```

PIN Entry:
```jsx
<AppInput
  label="Enter PIN"
  pin
  maxLength={4}
  keyboardType="number-pad"
  value={pin}
  onChangeText={setPin}
/>
```

Error Handling:
```jsx
<AppInput
  label="Email"
  error={errors.email}
  value={email}
  onChangeText={setEmail}
/>
```

MIGRATION PATH

If migrating from old components:

Old Component → New Component:
  AppButton → PrimaryButton (filled) or SecondaryButton (outlined)
  AppInput → AppInput (improved, same name)
  AppText → Typography (with variants)
  AppCard → LogoCard (for auth)
  ScreenWrapper → AuthLayout (for auth screens)

Simply replace imports:
```js
// Before
import { AppButton } from '@/components/common';
const MyBtn = () => <AppButton label="Click" />;

// After
import { PrimaryButton } from '@/components';
const MyBtn = () => <PrimaryButton label="Click" />;
```
