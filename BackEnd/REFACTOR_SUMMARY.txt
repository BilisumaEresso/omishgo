ROLE STRUCTURE COMPARISON

BEFORE (Enterprise Complex):
{
  type: String,
  verified: Boolean (default: false),
  subscription: String (default: "free"),
  active: Boolean (default: true)
}

AFTER (MVP Simple):
{
  type: String,                    // Role type
  status: "active",                // Status: active/pending/blocked
  subscriptionTier: "free"         // Subscription tier
}

Result: Cleaner, simpler, business-aligned ✅

USER REGISTRATION CHANGE

BEFORE (Role Selection):
POST /api/v1/auth/register
{
  "name": "John",
  "email": "john@example.com",
  "phone": "1234567890",
  "pin": "1234",
  "role": "farmer"  ← User had to choose
}

AFTER (Farmer-First):
POST /api/v1/auth/register
{
  "name": "John",
  "email": "john@example.com",
  "phone": "1234567890",
  "pin": "1234"
  // No role - automatically FARMER
}

Benefit: Simpler onboarding for farmers ✅
