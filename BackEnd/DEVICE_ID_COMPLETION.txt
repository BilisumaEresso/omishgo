ARCHITECTURE

Device Validation Flow:

  Login Request (email, pin, deviceId)
         ↓
  Validate Credentials (email/phone/PIN)
         ↓
  Call: validateDeviceForLogin(user, deviceId)
         ↓
    IF NODE_ENV="development":
      ✓ Pass (log for debugging)
         ↓
    IF NODE_ENV="production":
      Check: user.activeDeviceId === deviceId ?
      ✓ Yes → Pass
      ✗ No → Return device move prompt
         ↓
  IF Valid: Generate token, return user
  IF Invalid: Return error with device move option

MOBILE IMPLEMENTATION

Recommended Strategy:

Priority 1: Native device identifier (Device.deviceId from expo-device)
Priority 2: Installation ID (Device.installationId - more stable)
Priority 3: Persisted UUID in AsyncStorage (fallback)

Example: getStableDeviceId()

async function getStableDeviceId() {
  // Try native device ID
  if (Device.deviceId) return Device.deviceId;

  // Try installation ID
  if (Device.installationId) return Device.installationId;

  // Try persisted UUID
  const stored = await AsyncStorage.getItem('device_id');
  if (stored) return stored;

  // Generate and store new UUID
  const newId = uuidv4();
  await AsyncStorage.setItem('device_id', newId);
  return newId;
}

Send in every login request:
{
  email: "...",
  pin: "...",
  deviceId: await getStableDeviceId()
}
